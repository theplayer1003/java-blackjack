# 🚀 미션 - 블랙잭 (2단계: 비즈니스 로직 확장)

## ✏️ 과제 진행 요구 사항

- 1단계에서 구현한 도메인 모델(객체 구조)을 최대한 훼손하지 않고(OCP 준수), '베팅'과 '수익 계산'이라는 새로운 책임과 상태를 추가한다.

- 기존에 작성된 1단계의 테스트 코드가 깨지지 않도록 유지 보수하며, 추가된 요구사항에 대한 테스트 코드를 작성한다.

## 🚀 기능 요구 사항

기존 승패 판정 시스템에 자본의 흐름을 통제하는 베팅 시스템을 결합하여, 각 참가자별 최종 수익금을 산출한다.

### 베팅 및 수익 분배 규칙

- 게임 시작 시 각 플레이어는 자신의 배팅 금액을 정한다.

- 게임 종료 후 아래의 배당률(Odds) 규칙에 따라 베팅한 금액을 순차적으로 분배 및 정산한다.

    - 블랙잭 (초기 카드 2장의 합이 21):

        - 플레이어만 블랙잭인 경우: 딜러로부터 베팅 금액의 1.5배를 지급받는다.

        - 딜러와 플레이어 모두 블랙잭인 경우: 무승부 처리되어 베팅 금액을 돌려받는다(수익 0).

    - 일반 승리 (딜러 버스트 또는 딜러보다 높은 점수): 베팅 금액만큼의 수익을 얻는다.

    - 무승부 (동점): 베팅 금액을 돌려받는다(수익 0).

    - 패배 (플레이어 버스트 또는 딜러보다 낮은 점수): 베팅 금액을 모두 잃는다(수익 -베팅금액).

- 딜러의 최종 수익은 모든 플레이어들의 수익 합계의 반대 값(Zero-Sum)이 된다.

## 입출력 요구 사항

### 입력

- 각 플레이어별로 배팅 금액을 정수로 입력받는다. 음수 또는 문자가 입력될 경우 예외 처리 후 다시 입력받는다.

### 출력

- 최종 결과를 출력할 때 단순 승패(승/무/패) 대신 최종 수익 금액을 출력한다.

### 실행 결과 예시

```Plaintext
게임에 참여할 사람의 이름을 입력하세요.(쉼표 기준으로 분리)
pobi,jason

pobi의 배팅 금액은?
10000

jason의 배팅 금액은?
20000

(중간 게임 진행 출력 생략 - 1단계와 동일)

딜러 카드: 3다이아몬드, 9클로버, 8다이아몬드 - 결과: 20
pobi카드: 2하트, 8스페이드, A클로버 - 결과: 21
jason카드: 7클로버, K스페이드 - 결과: 17

## 최종 수익
딜러: 10000
pobi: 10000
jason: -20000
```

## 🎯 프로그래밍 요구 사항 (추가 제약)

- 1단계의 모든 프로그래밍 요구 사항을 유지하며, 다음의 객체 지향 제약 사항을 엄격히 추가 적용한다.

    - 모든 원시 값과 문자열을 포장(Wrap)한다.

        - 베팅 금액(Money), 점수(Score), 이름(Name) 등을 단순 int, String으로 사용하지 않고 의미를 부여한 값 객체(Value Object)로 설계한다.

    - 일급 컬렉션(First-Class Collection)을 사용한다.

        - 딜러와 플레이어가 가지는 List<Card> 컬렉션을 직접 외부로 노출하지 않고, 컬렉션만을 유일한 인스턴스 변수로 가지는 Hand 또는 Cards 객체를 생성하여 관련 비즈니스 로직(총점 계산,
          버스트 확인 등)을 내부로 캡슐화한다.
