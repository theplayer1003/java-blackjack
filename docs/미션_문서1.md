## 🚀 기능 요구 사항

블랙잭 게임은 딜러와 플레이어 중 카드의 합이 21 또는 21에 가장 가까운 숫자를 가지는 쪽이 이기는 게임이다. 상태와 행위를 캡슐화한 객체들을 도출하고, 이들의 협력을 통해 승패를 판정하는 핵심 도메인 로직을
구현한다.

### 카드 및 덱 (Deck) 관리

- 덱은 스페이드, 하트, 다이아몬드, 클로버 4가지 문양과 2~10, J, Q, K, A 숫자의 조합으로 총 52장의 유일한 카드로 구성된다.

- 게임 시작 시 카드를 무작위로 섞어 관리한다.

- 점수 계산 규칙:

    - 숫자 카드는 해당 숫자를 점수로 계산한다.

    - King, Queen, Jack은 각각 10으로 계산한다.

    - Ace는 1 또는 11 중 현재 카드 총합이 21을 초과하지 않으면서 가장 21에 가까워지는 최적의 숫자로 동적 계산되어야 한다.

### 게임 진행 규칙

- 초기 배분: 게임 시작 시 딜러와 각 플레이어는 덱에서 카드를 2장씩 무작위로 지급받는다.

- 플레이어 턴:

    - 플레이어는 카드 합이 21을 초과하지 않는 한, 계속해서 카드를 추가로 뽑을 수 있다(Hit).

    - 카드 합이 21을 초과하면 '버스트(Bust)' 상태가 되며 즉시 턴이 종료되고 패배한다.

- 딜러 턴:

    - 모든 플레이어의 턴이 종료된 후 진행한다.

    - 딜러의 카드 합이 16 이하이면 반드시 1장의 카드를 추가로 받아야 한다.

    - 딜러의 카드 합이 17 이상이면 더 이상 카드를 받을 수 없다(Stand).

### 승패 판정

- 딜러와 플레이어의 카드 합을 비교하여 최종 승패를 결정한다.

    - 플레이어가 버스트된 경우 딜러의 결과와 무관하게 플레이어 패배.

    - 플레이어가 버스트되지 않고 딜러만 버스트된 경우 플레이어 승리.

    - 양측 모두 버스트되지 않은 경우, 21에 더 가까운(점수가 더 높은) 쪽이 승리하며, 동점일 경우 무승부 처리한다.

## 입출력 요구 사항

### 입력

- 게임에 참여할 사람의 이름을 쉼표(,)를 기준으로 분리하여 입력받는다.

    - 이름은 공백일 수 없으며 중복될 수 없다.

    - 참가자는 최소 2명에서 최대 8명으로 제한한다.

- 카드를 추가로 받을지 여부를 입력받는다. (y 또는 n)

- 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException을 발생시키고, [ERROR]로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.

### 출력

- 게임 시작 시 초기 배분 상태를 출력한다. 딜러는 1장의 카드만 공개하고 플레이어는 2장 모두 공개한다.

- 딜러가 카드를 추가로 받았을 경우 해당 안내 메시지를 출력한다.

- 게임 종료 시 딜러와 모든 플레이어의 최종 카드 목록과 계산된 점수를 출력한다.

- 최종 승패 결과를 계산하여 출력한다.

### 실행 결과 예시


## 🎯 프로그래밍 요구 사항

- JDK 21 버전에서 실행 가능해야 한다.

- 프로그램 실행의 시작점은 Application의 main()이다.

- build.gradle.kts 파일은 변경할 수 없으며, 제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않는다.

- 자바 코드 컨벤션을 지키면서 프로그래밍한다.

- indent(인덴트, 들여쓰기) depth를 2가 넘지 않도록 구현한다. 1까지만 허용한다.

- 3항 연산자와 else 예약어를 쓰지 않는다.

- 함수(또는 메서드)의 길이가 10라인을 넘어가지 않도록 구현하며, 한 가지 일만 하도록 작게 만든다.

- 딜러와 플레이어에서 발생하는 중복 코드를 상속이나 인터페이스를 활용한 다형성으로 제거해야 한다.

- 입출력을 담당하는 클래스(UI 로직)를 핵심 도메인 로직과 명확히 분리하여 구현한다.

- 도메인 로직에 대한 단위 테스트를 JUnit 5와 AssertJ를 활용하여 100% 검증한다. (UI 로직 제외)

- 무작위로 카드를 추출하는 비결정적 로직(Random)은 외부에서 주입(DI)받도록 설계하여, 테스트 환경에서 결과를 통제할 수 있어야 한다.
